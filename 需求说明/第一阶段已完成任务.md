截至目前，第一阶段（MVP）已完成的修改可以总结为这几块：

1. 后端数据层与迁移已落地  
- 已新增事件相关表迁移：`010_add_market_events`（`market_events` / `market_event_articles` / `symbol_event_links`）。  
- 已新增补充迁移：`011_add_article_snapshot_fields`，为 `market_event_articles` 增加新闻快照字段（`title/article_url/source_name/published_at`）。  
- 已实现事件数据库访问模块（事件列表、热点、详情、按 symbol 查询、marker 查询、AI 解读回填）。

2. 事件聚合服务已上线（后台循环）  
- 已新增 `EventService`，复用新闻源做 24h 聚合，5 分钟周期运行。  
- 聚类规则已实现：ticker 交集 + 时间窗 + 标题 Jaccard；并生成稳定 `event_id`。  
- 已实现 importance score 计算、事件入库 upsert、文章关联入库、symbol 链接入库。  
- 应用生命周期已接入启动/关闭（跟随服务进程）。

3. 后端 API 已补齐  
- 新增事件路由：  
  - `GET /api/v1/events`  
  - `GET /api/v1/events/hot`  
  - `GET /api/v1/events/{event_id}`  
  - `GET /api/v1/events/by-symbol/{symbol}`  
  - `POST /api/v1/events/{event_id}/interpret`  
- 新闻解读接口已新增：`POST /api/v1/news/{article_id}/interpret`  
- MarketView marker 接口已新增：`GET /api/v1/market-data/stocks/{symbol}/events`  
- 新增/扩展了对应 Pydantic 模型（Event 列表、详情、marker、interpret 请求/响应）。

4. 前端 Dashboard 已事件化  
- `useDashboardData` 已接入事件流与热点事件数据。  
- `NewsFeedCard` 已支持双模式：`Events` / `Raw News`。  
- Events 模式下已支持二级切换：`Hot Events`（默认）与 `All Events`。  
- 已实现 `EventDetailModal`（摘要、symbols、关联新闻、AI Interpret、跳转 MarketView）。  
- 已支持 Dashboard 深链 `?event=<event_id>` 自动打开事件详情弹窗。

5. 前端 MarketView 已完成事件联动  
- 已接入 symbol 事件 marker 拉取并叠加到图表。  
- marker 点击可弹事件信息卡。  
- 弹窗已支持 `View Event` 跳转 Dashboard 深链（形成 MarketView -> Dashboard 闭环）。

6. Docker 与运行稳定性改造已完成关键项

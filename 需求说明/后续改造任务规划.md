# LangAlpha 后续改造任务规划（基于第一阶段已完成）

## 1. 目标与原则

- 以当前第一阶段已完成能力为基线，对齐《完整需求说明》补齐后续任务。
- 实施顺序遵循业务价值：先资讯增强，再日报能力，再研报MVP，最后高阶功能。
- 保持第一阶段接口兼容，不回退现有 `Events -> EventDetail -> MarketView marker` 闭环。

---

## 2. 现状与差距

### 2.1 已完成（第一阶段）

- 事件化资讯链路：事件聚合、入库、事件流、热点、详情、AI解读。
- MarketView 联动：事件 marker、事件跳转、Dashboard 深链回显。
- 运行保障：容器启动自动迁移、数据库迁移重试、核心单元测试补齐。

### 2.2 仍需补齐（来自完整需求）

- 资讯侧：按板块/主题筛选、资讯热榜治理、单条资讯一键提问、早报/晚报、7x24快讯、大事地图。
- 研报侧：研报数据接入、研报列表/详情、看涨看跌观点提取、摘要与逻辑、观点一致性、历史观点变化。
- 通用侧：效果评估指标、可观测性、合规开关与免责声明、更完整回归测试矩阵。

---

## 3. 分阶段实施计划

## 3.1 Phase 2A：资讯增强（P0，2个迭代）

- 新增资讯标签体系：板块、主题、地域、品种关联标签。
- 新增资讯筛选能力：按板块/主题/品种聚合查询与前端筛选入口。
- 新增热榜治理：热度、时效、来源多样性综合排序。
- 新增“单条资讯一键提问”：从资讯卡片直达 AI 对话并附带上下文。
- 新增 7x24 快讯展示模式（与 `Events/Raw News` 并列或切换）。

**交付标准**
- 用户可在资讯页完成：筛选 -> 查看详情 -> 一键提问 -> 返回列表。
- 热榜结果具备可解释排序字段（importance/recency/source_count）。

## 3.2 Phase 2B：日报能力（P1，1个迭代）

- 新增早报/晚报生成任务（定时生成、可回看）。
- 报告内容最小闭环：市场概览、热点事件、关键品种、风险提示。
- 先上线文本版；TTS 作为配置开关，默认关闭。

**交付标准**
- 可按日期读取晨报/晚报。
- 报告生成失败不影响主资讯流程，支持重试与状态追踪。

## 3.3 Phase 3：研报MVP（P0，2-3个迭代）

- 建立研报数据模型与入库：研报主表、机构表、观点表、品种映射表。
- 上线研报列表页与详情页。
- 上线研报 AI 能力：摘要、核心观点、看涨/看跌标签、研究逻辑提炼。
- 上线“同品种观点一致性”与“同机构历史观点变化”最小可用版本。
- 上线“品种多空报告查询”接口与页面入口。

**交付标准**
- 用户可完成：研报列表 -> 研报详情 -> AI解读 -> 一致性查看。
- 观点标签有可追溯来源（规则/模型结果）与置信度字段。

## 3.4 Phase 4：高阶能力（P1/P2，1-2个迭代）

- 大事地图（地域事件聚合可视化）。
- 观点命中率/回溯评估（如合规允许）。
- 自定义资讯看板（低优先级，可延后）。

## 4. 接口与数据规划（后续新增）

## 4.1 资讯侧接口

- `GET /api/v1/news/hot-rank`
- `GET /api/v1/news/by-sector/{sector}`
- `GET /api/v1/news/by-topic/{topic}`
- `POST /api/v1/news/{article_id}/ask`

## 4.2 日报接口

- `GET /api/v1/briefs/daily?date=YYYY-MM-DD&type=morning|evening`
- `POST /api/v1/briefs/{brief_id}/interpret`

## 4.3 研报接口

- `GET /api/v1/reports`
- `GET /api/v1/reports/{report_id}`
- `GET /api/v1/reports/by-symbol/{symbol}`
- `GET /api/v1/reports/consensus/{symbol}`
- `POST /api/v1/reports/{report_id}/interpret`

## 4.4 关键类型（新增）

- `NewsTag`
- `NewsHotRankItem`
- `DailyBrief`
- `ResearchReport`
- `ReportViewpoint`
- `ConsensusSnapshot`

## 5. 测试与验收

## 5.1 自动化测试

- 后端：新增路由、排序/聚合规则、定时任务、异常回退。
- 前端：筛选交互、热榜展示、一键提问、日报阅读、研报详情与一致性展示。
- 回归：第一阶段既有 `events`、`market-data events` 全量通过。

## 5.2 Docker 验收（最终标准）

- 容器启动自动迁移通过（`alembic upgrade head` 自动执行）。
- 资讯闭环通过：筛选 -> 详情 -> 提问 -> 返回。
- 日报闭环通过：按日期读取晨报/晚报。
- 研报闭环通过：列表 -> 详情 -> 解读 -> 一致性/多空查询。

---

## 6. 风险与默认策略

- 默认先文本闭环，TTS/地图后置，避免首轮复杂度过高。
- 观点一致性先规则+轻量模型，后续再做精度迭代。
- 对可能触及合规的“建议性表达”加免责声明与配置开关。
- 保持现有第一阶段接口与页面行为向后兼容。
